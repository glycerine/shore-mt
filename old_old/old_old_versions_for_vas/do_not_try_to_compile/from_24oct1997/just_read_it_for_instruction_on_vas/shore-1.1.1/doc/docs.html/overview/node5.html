Status of Release

As mentioned in footnotes sprinkled throughout this document, not all of the features of Shore are supported in this release. The release of Shore is intended for initial SDL-level application development work. Also, applications are restricted to using a single server at this stage.

A long-term goal is that the Shore type system will reside in the server so that the server can make the strongest reasonable guarantees about the integrity of objects and their relationships. In this release, however, the entire type system is encapsulated in the language binding generated from an SDL type description. Consequently, Shore makes no guarantees about the integrity of any objects other than fundamental file system objects, such as directories, symbolic links, and the like.
